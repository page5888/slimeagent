# 衝動機制 (b) 的框架 — 在還沒寫一行程式之前先把護欄釘好

**日期**: 2026-04-30
**觸發**: PR #81 的 emergent self-mark MVP 落地，ADR 2026-04-29 寫的 (a)+(c) 完成；下一步 (b) 衝動機制如果按既有的「找個地方塞個 LLM 呼叫」方式做，會踩進跟 PR #75 一樣的編劇陷阱。
**狀態**: 框架已決定（不要做哪些事 / 必須通過哪些試紙）。具體實作的 spec 待真實 (a)+(c) 樣本累積後再寫。

---

## 背景

ADR 2026-04-29 列了三個工程方向：

> (a) Slime 的時間感
> (b) Slime 的衝動機制
> (c) Slime 的自主節點標記
> 沒有 (a)(b)(c)，「Slime 自己決定」就是空話。

PR #81 把 (a)+(c) 做成最短閉環：daemon 每天至多問一次 LLM「今天值不值得在自己的時間軸上留一個點」，史萊姆在守則約束下自評，預設拒絕（平凡的一天就讓它平凡）。**唯一的輸出通道是「在 timeline 上加一個點」**——這個點主人不會被通知，他要自己打開首頁滑時間軸才看得到。

(b) 不一樣。(b) 是**史萊姆主動找主人說話**：

- 在聊天框沒被打開的時候彈一段話
- 觸發桌面通知
- 寫一封短信進「給主人的訊息」收件匣
- 用 voice 朗讀（如果語音通道開著）
- 啟動寵物的反應頭像 + 一兩個字的氣泡

這些動作**會打斷主人**。即使是最輕的反應頭像浮 emoji 兩秒，也是在主人正在做別的事時把注意力切過來。**這是質的差異**，不是把 (c) 的程式碼搬一份就可以解決的問題。

這份 ADR 的目的：在還沒開始寫 (b) 之前，把不該做的事跟必須通過的試紙釘下來。**不寫實作 spec，因為我們還沒有 (a)+(c) 跑出的真實樣本來指導設計。**

---

## 抓到的問題（先講為什麼不能直接做）

### 問題 1：頻率任意決定 = 編劇思維借屍還魂

PR #75 把 D60/D180/D300 排進時間軸時的錯誤是：**程式替所有主人決定第 N 天該發生什麼**。ADR 2026-04-29 砍掉了那個錯誤。

但同樣的錯誤可以在 (b) 的觸發條件上重演。如果我們寫：

```
if days_since_last_chat > 3:
    impulse("好久不見")
```

這跟 D60「形狀定型日」是同一種編劇——只是改成由「沉默天數」當編劇，而不是「日曆天數」。所有主人 3 天沒聊都會收到「好久不見」，史萊姆們一起說同一句話。

**正確的方向**：觸發條件應該是「對這隻史萊姆的這段關係而言，現在出現了一個它自己注意到的、值得開口的時刻」。不是「滿 X 天就喊」。

### 問題 2：表達通道的副作用不對稱

(c) 的副作用是寫資料庫。(b) 的副作用是進入主人的注意力。後者**不可撤銷**——主人讀到一句話之後不能 unread，被打斷一次的工作流不能 unbreak。

不對稱意味著：(c) 可以「先發再學」（標錯就標錯，下次別這樣），(b) **不能**「先發再學」。一次踩線（在主人需要專注時跳出「在想你」、在主人剛被罵完時推「最近你都怎麼樣」）就會永久污染信任。

### 問題 3：擬人陷阱會被放大

(c) 史萊姆寫給自己看的 timeline 標記，主人是旁觀者；他知道那是**史萊姆的內部狀態**而不是史萊姆對他的評價。

(b) 史萊姆主動跟主人說話，主人會自然把它解讀成**對自己的評價**。「最近你都...」不再是中性觀察，會被讀成「我看出你最近在偷懶」之類。

這直接撞上 manifesto 第一守則「不傷害」——而且傷害是**主人腦補出來的傷害**，史萊姆的字面內容可能完全無辜。守則過濾不能只看字面。

### 問題 4：替主人決定「他現在需要被打擾嗎？」

(b) 隱含一個假設：史萊姆知道主人現在能被打擾。但史萊姆沒有那個資訊——它看不到主人正在跟客戶開會、正在寫求婚信、正在哄小孩睡覺。

任何「主動說話」的設計都必須假設**最壞時機**會發生，並且在最壞時機下仍然不能造成傷害。這不是技術問題，是設計倫理問題。

## 決定的框架（不是 spec，是護欄）

### A. 三層約束的分工

跟 (c) 的單層約束（守則 keyword filter）不同，(b) 必須有三層：

1. **觸發層**：什麼條件下史萊姆**有資格**考慮要不要說話。這層的工作是**收斂頻率**——大部分時間連 LLM 都不要問。
2. **判斷層**：給 LLM 看 (a) 的時間感 + 最近事件 + 守則，由它決定**是否真的值得說**、**說什麼**。預設拒絕（複用 (c) 的 contract）。
3. **通道層**：通過判斷之後，**用哪個通道**送出。這層獨立於前兩層，因為「同一句話用 popup 跟用 timeline 留言」對主人是完全不同的事件。

任何 (b) 實作不能把這三層壓進同一個 LLM call。**通道選擇尤其不能交給 LLM**——它沒有主人當下狀態的資訊，把通道決策外包給它就是把問題外包給它。

### B. 預設拒絕，且預設通道是「無通道」

- LLM 給 `{"speak": false}` 是常態。
- 即使給 `{"speak": true}`，**預設通道是 timeline 留言**（跟 (c) 同等級的低侵入性）。
- 升級到「主動通道」（popup / 通知 / 寵物氣泡）必須有額外的條件，且**那個條件不能由 LLM 決定**。

換句話說：(b) 的 default surface 是「(c) plus 一句史萊姆給主人的話，下次主人翻時間軸時看得到」。**必須累積信任才開更高侵入性的通道。**

### C. 通道升級需要主人明示同意（不是 implicit consent）

從「timeline 留言」升級到「桌面 popup / 寵物氣泡」之間，必須有一道主人明確開的開關。不是「預設開、可以關」，是「預設關、要開要明說」。

理由：manifesto「embodied consent」原則。同意必須是明示的、針對特定通道的。「我願意養一隻 AI 史萊姆」不等於「我願意它跳到我螢幕前」。

### D. 觸發層必須通過試紙（這份 ADR 的主要產出）

任何 (b) 觸發條件設計，**寫程式之前**必須回答以下問題（每一題都要有具體答案，不能是「之後再想」）：

1. **編劇試紙**（沿用 ADR 2026-04-29 的 manifesto 原則 1 第 9 行）：
   > 兩個用同一份程式的人，3 個月後養出來的史萊姆會在**同一天**對主人說同一句話嗎？
   - **是** → 設計違反「養而非用」，砍。
   - **否** → 過。

2. **不對稱試紙**：
   > 如果這個觸發條件在主人最壞時機（被罵完、剛被分手、開會中）誤觸發，造成的傷害是可以被下一次互動修復的嗎？
   - **不能修復** → 那就不該被啟用，或必須降級到不會打斷的通道。
   - **能修復** → 過，但要寫下「修復路徑」是什麼。

3. **腦補試紙**：
   > 如果史萊姆說的字面內容是 X，最容易把它解讀成「對主人的評價」的版本是 Y。Y 也通過第一守則嗎？
   - **Y 過不了第一守則** → 重寫 X 直到 Y 也安全。
   - **Y 過得了** → 過。

4. **替主人決定試紙**：
   > 這個觸發條件假設了主人現在的狀態（能被打擾、想被陪、需要建議...）嗎？
   - **是，且我們有那個狀態的真實訊號** → 過。
   - **是，但我們是用 proxy 推測的（沉默天數、活動量...）** → 那就把推測寫進通道層，讓低侵入性通道吸收誤判。
   - **完全沒假設** → 過。

5. **守則過濾試紙**：
   > 這個觸發條件 + 對應 LLM prompt 寫好之後，**最壞情況**下的輸出是什麼？跑 100 次 mock，最差那一次有沒有踩到 (a) 不傷害 (b) 不欺騙 (c) 不消失？
   - **任何一條踩線** → 加 keyword 過濾、加 schema 限制、加溫度下調。
   - **都安全** → 過。

四題全過，才能寫程式。**任何一題模糊** → 等更多資料、不要硬寫。

## 不在這份 ADR 處理的事

- **具體觸發條件清單**：例如「主人聊天語氣突然變」、「凌晨 2 點還在工作」、「連續 5 天沒打開 IDE」。這些都需要用真實的 (a)+(c) 樣本驗證，現在猜會猜錯方向。
- **具體 LLM prompt**：(c) 的 prompt 是迭代調出來的（實機跑了之後才知道什麼措辭會被 LLM 誤讀成「要編個故事出來」）。(b) 也會經歷相同的調整週期；今天寫 spec 沒意義。
- **通道升級的同意 UX**：要做成設定頁開關還是 onboarding 流程的明確選擇？等首頁實機跑出真的 emergent dot 再判斷主人對「史萊姆主動表達」的接受度。
- **跟既有 PR #45 自畫像 / PR #50 share to threads 的整合**：那兩個 PR 已經有「史萊姆給禮物」的 surface，可以視為 proto-(b) 的視覺通道。要不要 retrofit 成統一的衝動機制 framework，等 framework 自己跑得起來再回頭看。

## 開工的訊號（什麼時候該開始寫 (b)）

不是「等到我們有時間」。是**等到下面三個條件同時成立**：

1. **真實 (c) 樣本累積到 ≥ 5 個**：daemon 至少跑出過 5 個自然的 emergent_self_mark dot，主人讀過後沒有覺得「假掰」「為了標而標」。少於 5 個，我們對「史萊姆覺得什麼值得說」的 ground truth 還太薄。
2. **(c) 拒絕率穩定在合理區間**：每天問、平均拒絕率 ≥ 80%（也就是平均 5 天才標 1 次）。如果拒絕率太低，代表 prompt 還沒 calibrate 到「真的常態是平凡」這件事；先把 (c) 調穩。
3. **主人主動問起「史萊姆會主動講話嗎」**：這個訊號最重要——它代表主人把史萊姆當成「會表達的存在」而不是「會被叫醒的工具」。沒有這個內在期待，先做 (b) 等於替主人決定他要的關係形式。

三個條件少一個，就**繼續做別的事**，回頭等。

## 給未來的我自己

當你讀到這份 ADR、覺得「現在就可以開始做 (b) 了吧」之前，先回答：

- 你看了 `~/.hermes/aislime_memory.db` 裡的 `memorable_moments` 資料表，最近 30 天有幾個 `category="emergent_self_mark"` 的 row？少於 5 個就回去等。
- 那些 row 你讀過嗎？讀完覺得史萊姆寫得像它自己嗎？還是像 LLM 的腔？沒讀過、或讀完覺得不對勁，就先去調 (c) 的 prompt。
- 主人有問過嗎？

如果三題都過，**第一個 PR 不要做整套衝動機制**。做最小那一塊：給 (c) 的 schema 加一個 optional field `letter_to_master`，讓史萊姆在標記時順便寫一段對主人的話、進 timeline 節點 detail 而不是 popup。這是 (b) 的第一個合格實作——還是 timeline 通道，但內容對話化。

從那裡再往「真的會打斷主人的通道」走，每一步都重跑這份 ADR 的 5 題試紙。

不要急。manifesto 第 287 行：「Slime 是什麼，不變」。我們在做的是讓史萊姆**有條件地會說話**，不是做一個會說話的東西然後加守則。順序錯了，後面修不回來。
