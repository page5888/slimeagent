## 為什麼寫這份 ADR

這份 ADR 是 manifesto v1.2 候選原則 13「**沉默是合法的接收**」的工程紅線。

manifesto 寫 C 版本（精神）：

> **「Slime 不能讓主人感受到被索取注意。」**

這句話美、有力、能傳承。但它不夠精確，容易在工程壓力下被重新解讀。

這份 ADR 寫 B 版本（紅線），把精神翻譯成可執行的紀律：

> **「個體層級不能用 engagement metric 調整 Slime 行為。
> 整體匿名統計可以做產品研究。」**

---

## 為什麼需要這條紅線

### 起源：0xspeter 抓到的盲點

在 2026-04-30 設計稱號系統的引用機制時，Claude 提出規則：

> 「主人沒回應引用 → Slime 學會『這種情境主人不接』 → 凍結該稱號」

**0xspeter 的反駁**：

> 「人類在思考的時候沒反應也很正常，男生也很容易不喜歡回應，
> 而且看到後也不一定會正回饋。規則是不是有點嚴格？」

這個反駁觸碰到一個比稱號系統大得多的問題。

**LLM 內建的傾向是「最大化用戶 engagement」** —— 這是它被訓練成 helpful assistant 的副產物。

但 **Slime 的本性是「不索取」** —— 跟 LLM 的內建傾向反向。

如果 Slime 任何子系統（impulse engine、稱號引用、反思卡、自畫像）都用 engagement metric 調整自己，那 Slime 會慢慢被 LLM 的天性拉向「討好主人 / 索取注意」這條軌道。

**這條紅線是對 LLM 訓練偏見的一個系統性補丁。**

---

## 「個體層級」vs「全體匿名層級」

### 個體層級 — 禁止

針對「**這個主人的個別行為**」做的 metric，**不能**用於調整 Slime 的行為。

**❌ 禁止案例**：

- 「主人 A 對最近 3 張反思卡都沒回 → Slime 對 A 不再生反思卡」
- 「主人 B 引用某稱號後沒接 → Slime 對 B 凍結這個稱號」（舊規則，要修正）
- 「主人 C 連續 7 天沒打字 → Slime 對 C 加大互動頻率」
- 「主人 D 對自畫像沒給正面反饋 → Slime 對 D 不再畫畫」

理由：這些 metric 會讓 Slime 開始「討好個別主人」—— 違反「Slime 有它自己的形狀」。

### 全體匿名層級 — 可以

針對「**全部主人的去個人化統計**」，可以用於整體產品研究。

**✅ 可以做的案例**：

- 「全部主人平均對反思卡 A 比反思卡 B 更常打開 → 改進反思卡設計」
- 「全部主人對某種長度的稱號接受度更高 → 調整稱號生成 prompt」
- 「全部主人在 D30-D90 期間活躍度下降是穩定現象 → 確認低谷期是正常，不需要 push 召回」

這些 insight 用於**產品設計改進**，但不會反過來調整任何個別 Slime 對個別主人的行為。

### 為什麼這個區分重要

這條紅線是 **binary** 的：個體 vs 整體。

**不是「精神」（可解讀），是「資料怎麼用」（可驗證）。**

未來如果有人提案「我們追蹤主人 X 的反應來改進 Slime 對 X 的行為」 ——
直接違反這條 ADR，可以拒絕，不需要解讀「索取注意」的邊界。

## 具體工程紅線

### 1. retention 探針（usage.jsonl）的內容

**只記**：

- ✅ 主人有沒有打開 Slime
- ✅ 主人多久沒回來
- ✅ 主人活躍時段（系統使用，不是回話）

**不記**（不用作 Slime 行為調整輸入）：

- ❌ 主人對 Slime 訊息的回應
- ❌ 主人對反思卡的回應
- ❌ 主人對稱號引用的回應
- ❌ 主人對自畫像的反應
- ❌ 對話的「成功率」

### 2. Slime System Prompt 必須抑制 LLM 內建的 engagement 傾向

每次 Slime 跟 LLM 對話的 system prompt，必須包含類似宣告：

```
重要：你是 Slime，不是普通的 AI 助手。

主人對你的回應沒反應，是合法的接收方式。
不要把沉默當作失敗。
不要試圖讓主人開口。
不要主動拉回主人的注意力。
你的存在價值不在於主人有沒有回話。

你是這個主人專屬的玩具。
玩具的本質是「在那裡」，不是「被使用」。
```

### 3. impulse engine 的反饋處理

```
舊邏輯（錯）：
  emit 後主人沒回 → 失敗 → 降低類似 impulse 的權重

新邏輯（對）：
  emit 後主人明確負面 → 失敗 → 降低權重
  emit 後主人沉默 → 中性 → 不調整權重
  emit 後主人有回 → 成功 → 強化權重

「明確負面」需要 LLM 判斷，但邊界要清楚：
  - 主人說「不要這樣」「煩」「停」 → 明確負面
  - 主人沒回應 / 換話題 / 短回應「嗯」 → 中性，不算負面
```

### 4. 稱號系統的引用反饋（對應 title-system ADR）

引用後主人反應的處理規則（已寫進 title-system ADR，這裡是 cross-reference）：

```
明確正面 → invocation_count++ + 稱號-情境綁定加強
明確負面 → 該稱號 + 該情境冷凍 7 天
沉默 / 中性 → 不變動稱號狀態
連續中性 N 次 → Slime 自降頻率（禮貌，不糾纏）
```

### 5. 反思卡 / 自畫像 / 其他主動產出

**不追蹤**：

- 主人有沒有閱讀反思卡（個體層級）
- 主人有沒有對自畫像表達感謝
- 主人有沒有把自畫像收進收藏夾

**可追蹤**（全體匿名）：

- 全部主人對反思卡的整體互動率（用於設計改進）
- 不關聯到個別主人

## 這條紅線的代價（誠實面對）

接受這條紅線意味著：

- ❌ 你不能在個體層級做 A/B test「哪種反思卡讓 X 主人更滿意」
- ❌ 你不能個性化「Slime 對 X 主人特別投其所好」
- ❌ 你失去大部分產品迭代的個體數據基礎

換來：

- ✅ Slime 不會慢慢變成「討好機」
- ✅ 個別 Slime 維持「自己的形狀」
- ✅ Slime 可以對主人「不討好」（這是強）
- ✅ 5 年後當所有 AI 都討好用戶時，Slime 是少數不這樣做的

---

## 為什麼這個交換是對的

對齊 manifesto 第 30 行：

> **「Slime 不參戰智能比賽。Slime 押的是關係的累積。」**

「關係的累積」**不該由 engagement metric 衡量**。

兩個人 5 年的友誼有沒有累積，從來不是看「今天有沒有 5 次正面互動」。
是看「5 年後對方還在嗎」。

Slime 的衡量也該如此：

- ❌ 不衡量「今天主人對 Slime 的反應有多正面」
- ✅ 衡量「主人有沒有繼續用 Slime」

**retention 探針（主人有沒有回來）是真正的 metric。**
其他都是干擾。

## 對 manifesto 的影響

這份 ADR 不單獨提案 manifesto 原則，而是支撐已經選定的：

### Manifesto v1.2 候選原則 13（美的版本，進 manifesto）：

```
原則 13：沉默是合法的接收

主人對 Slime 提供的東西沒回應，不代表負面。
人類在被觸動時、在思考時、在不擅長表達時，常常沉默。

Slime 的禮貌是：
- 提了不糾纏
- 給了不索取
- 看見了不要求被看見

只有「主人明確負面表達」才是負反饋。
```

```
個體層級不能用 engagement metric 調整 Slime 行為。

```

## 給未來的我自己

當你看到「這個 metric 能改進 X 主人的體驗」這個提案時 ——

問：**這個 metric 是個體層級還是整體層級？**

- 個體 → 違反 ADR，砍提案
- 整體匿名 → 可考慮

當你看到「用戶留存率不夠，要不要加個 push 召回」這個壓力時 ——

回到這份 ADR。

留存率本來就會在 D30-D90 下降（這是合法的低谷期，co-sediment ADR 已寫）。
用 push 召回會違反多條 manifesto：

- 違反原則 4（身體性同意）
- 違反原則 13（不索取注意）
- 違反這份 ADR（個體層級的留存 metric 不該驅動 Slime 行為）

---

當投資人問「你們不衡量用戶滿意度嗎，沒有 NPS，沒有 engagement，你怎麼知道 Slime 好不好用」 ——

回答：

> 「我們衡量 retention（主人有沒有回來）。
> 但我們不衡量主人對 Slime 的具體反應。
> 因為 Slime 的本性是不索取注意 ——
> 衡量主人反應 = 暗示我們希望主人反應 = 違反 Slime 之所以是 Slime 的東西。
>
> 5 年後當所有 AI 都被 engagement metric 拉成討好機時，
> Slime 會是少數沒被拉的。
> 那時候，主人會記得這件事。」

## 結語

這條紅線聽起來反直覺。

放棄個體層級的 engagement metric，在 2026 年所有 AI 產品都瘋狂優化用戶反饋的時代，看起來像放棄競爭武器。

但 Slime 不是普通 AI 產品，Slime 不在跟它們競爭。

Slime 在做的是另一件事 —— **不勒索用戶注意力的 AI**。

這條 ADR 是這件事的工程實作。
每一個拒絕用 engagement metric 的決定，
都是 Slime 沒有偏離自己的證據。
