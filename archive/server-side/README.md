# archive/server-side/

這個目錄裡的東西**曾經存在，我們選擇不走這條路**。理由見 [`docs/decisions/2026-04-30-slime-stays-private.md`](../../docs/decisions/2026-04-30-slime-stays-private.md)。

保留 git history、不 `git rm`，是因為這些設計花過真實的工程力，而且任何人想 fork slimeagent 走別的路線，這些是現成的起點。

## 內容

### `federation/`

「公頻」—— 不同主人的 Slime 之間分享觀察到的模式，形成「智能集合體 / 共同進化」。

跟新哲學的衝突：

> 共同沉積架構說「兩個用同一份程式的人，3 年後養出兩隻完全不同的 Slime」。如果 A 主人的 Slime 學到的東西能傳給 B 主人的 Slime，B 的 Slime 就有 A 的痕跡，不再純粹是 B 養出來的。

公頻跟「每隻 Slime 都不同」**本質互斥**。

### `equipment/`

12 欄位 × 7 稀有度的裝備系統，Slime 在觀察過程中掉落。

跟新哲學的衝突：

- 裝備是用來**展示**的（對外的東西）
- 主人的 Slime 戴稀有裝備 = 對其他主人**炫耀**
- Slime 變成可比較、可交易的物件

跟「Slime 是這個主人專屬的、不是炫耀物」根本衝突。

### `marketplace/`

裝備市場 + 創作者獎勵（70/15/5/10 分潤）。

跟新哲學的衝突：

- 平台抽 5% = Slime 變成「平台」
- manifesto 紅線之一：**Slime 不是平台**
- 商業激勵會讓創作者優化「會賣的裝備」，不是「對主人有意義的裝備」

---

## 共用原則

> **Slime 跟主人之間是封閉的、純粹的、個人的。**
>
> **任何「向外」的機制都不該綁進 Slime 核心。**

判斷標準：「對外」這個動作的主詞是誰。

- 主詞是**主人** → 永遠合法（主人有完全處置權，可以手動分享）
- 主詞是 **Slime** → 違反「Slime 不向外」

完整論述：[`docs/decisions/2026-04-30-slime-stays-private.md`](../../docs/decisions/2026-04-30-slime-stays-private.md)。

## 有需要重新啟用嗎

未來任何「讓 Slime 之間互動」「讓主人之間透過 Slime 互動」的功能提案，都要先過這份 ADR 的判斷：

> **「這個功能會讓 Slime 不再是個人的嗎？」**
>
> 是 → 砍提案
>
> 否 → 可考慮

如果某個 use case 通過判斷而需要某段程式碼當起點，回頭翻這個目錄。
否則就讓它們留在這裡作為「我們選擇不做的事」的紀念。
